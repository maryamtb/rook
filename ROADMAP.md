## Roadmap

What's coming to Rook. For shipped releases see [CHANGELOG.md](./CHANGELOG.md).

## now

**code blocks v2.** code blocks are central to how rook is used, so the next release is dedicated to making them feel solid:

- durable persistence across quit and reopen
- sharper language auto-detection on short and ambiguous snippets
- a selection menu that always tracks the cursor, even in tall blocks
- ⌘S to commit a checkpoint to history, alongside the silent autosave

## next

- auto-updates via sparkle
- markdown support
- broader syntax coverage for common cli tools (aws, docker, kubectl) inside bash blocks

## later

a few ideas: links between notes, urls v2, mermaid/diagrams, Pro version
